## A

### Assessment
Avaliação de conformidade ou maturidade baseada em um framework específico. Um assessment consiste em responder perguntas e fornecer evidências.

### AI Assistant
Assistente de inteligência artificial integrado ao TrustLayer que ajuda usuários a responder perguntas, interpretar resultados e sugerir melhorias.

### Audit Log
Registro imutável de todas as ações realizadas no sistema, incluindo quem fez o quê, quando e de onde.

### Audit Trail
Sequência cronológica de registros de auditoria que permite rastrear todas as modificações em um dado.

## C

### Compliance
Conformidade com regulamentos, normas ou políticas internas. O TrustLayer ajuda a medir e gerenciar compliance.

### Control
Medida de segurança ou processo implementado para mitigar riscos. Controles podem ser técnicos, administrativos ou físicos.

### CIS Controls
Framework de segurança cibernética desenvolvido pelo Center for Internet Security com controles priorizados.

## D

### Dashboard
Painel visual que apresenta métricas, gráficos e indicadores de forma consolidada. TrustLayer oferece dashboards Executive, GRC e Specialist.

### Domain
Área de governança específica (ex: AI Governance, Cloud Security, Data Privacy). Cada domain pode ter múltiplos frameworks.

### Draft
Status de um assessment que ainda está sendo preenchido e não foi submetido.

## E

### Evidence
Documentação ou prova que suporta uma resposta em um assessment. Pode ser texto explicativo ou arquivo anexado.

### Executive Dashboard
Dashboard simplificado com métricas de alto nível para gestores e executivos.

## F

### Framework
Conjunto estruturado de controles, práticas ou requisitos usados para avaliar conformidade. Exemplos: NIST CSF, ISO 27001, CIS Controls.

## G

### Gap
Lacuna identificada quando uma resposta indica não conformidade ou conformidade parcial com um controle.

### Gap Analysis
Análise das lacunas identificadas em um assessment, geralmente com recomendações de remediação.

### GRC
Governance, Risk and Compliance - Abordagem integrada para gerenciar governança corporativa, riscos e conformidade.

### GRC Dashboard
Dashboard detalhado para profissionais de GRC com métricas técnicas e operacionais.

## H

### Heatmap
Visualização que usa cores para indicar intensidade ou status. No TrustLayer, usado para mostrar scores por categoria.

## I

### ISO 27001
Norma internacional para sistemas de gestão de segurança da informação (SGSI).

## L

### LGPD
Lei Geral de Proteção de Dados - Legislação brasileira de proteção de dados pessoais.

## M

### Maturity Level
Nível de maturidade que indica quão desenvolvido está um processo ou controle. Geralmente varia de 1 (inicial) a 5 (otimizado).

### MFA (Multi-Factor Authentication)
Autenticação multifator que requer mais de uma forma de verificação (senha + código, por exemplo).

### Module
Componente funcional do TrustLayer (ex: Assessments, Dashboards, Reports). A arquitetura modular permite customização.

## N

### N/A (Not Applicable)
Resposta que indica que um controle não se aplica ao contexto da organização.

### NIST
National Institute of Standards and Technology - Instituto americano que desenvolve frameworks de segurança.

### NIST CSF
NIST Cybersecurity Framework - Framework de segurança cibernética amplamente adotado.

### NIST AI RMF
NIST AI Risk Management Framework - Framework para gerenciamento de riscos de inteligência artificial.

## O

### Organization
Entidade principal no TrustLayer. Cada organização tem seus próprios usuários, assessments e configurações.

## P

### Partial
Resposta que indica conformidade parcial com um controle. Geralmente conta como 50% do score.

### Permission
Autorização para realizar uma ação específica no sistema. Permissões são agrupadas em roles.

## R

### RBAC (Role-Based Access Control)
Controle de acesso baseado em papéis. Usuários recebem roles que determinam suas permissões.

### Report
Documento gerado a partir de dados do TrustLayer, podendo ser exportado em vários formatos (HTML, Excel, PDF).

### Risk
Possibilidade de um evento negativo ocorrer. O TrustLayer ajuda a identificar e gerenciar riscos.

### RLS (Row Level Security)
Segurança em nível de linha no banco de dados que garante isolamento de dados entre organizações.

### Role
Papel atribuído a um usuário que define suas permissões. Roles disponíveis: Admin, Manager, Analyst, Auditor, Viewer.

## S

### SAML
Security Assertion Markup Language - Protocolo usado para Single Sign-On (SSO).

### Score
Pontuação calculada com base nas respostas de um assessment. Geralmente expressa como porcentagem (0-100%).

### Specialist Dashboard
Dashboard técnico com métricas detalhadas para especialistas em domínios específicos.

### SSO (Single Sign-On)
Autenticação única que permite usar credenciais corporativas para acessar o TrustLayer.

### Status
Estado atual de um assessment: Draft (em andamento), Completed (finalizado), Archived (arquivado).

## T

### Tenant
Instância isolada de uma organização no sistema multi-tenant do TrustLayer.

### Theme
Tema visual da interface. TrustLayer suporta múltiplos temas incluindo modo escuro.

### TOTP
Time-based One-Time Password - Código temporário gerado por apps autenticadores (Google Authenticator, etc.).

### Trend
Tendência de um indicador ao longo do tempo. Dashboards mostram trends de scores.

## U

### User
Pessoa com acesso ao TrustLayer. Cada usuário pertence a uma organização e tem uma role atribuída.

## W

### WebAuthn
Padrão de autenticação que permite usar chaves de segurança físicas (YubiKey, etc.) como segundo fator.

### Widget
Componente visual em um dashboard que exibe uma métrica ou gráfico específico.
